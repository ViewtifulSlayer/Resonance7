# Workspace MCP Servers

Quick reference for **framework** MCP servers shipped with Resonance7. For launch config see [`.cursor/mcp.json`](../../.cursor/mcp.json) (gitignored; generated by `setup_database.py`).

> **Local copy:** `library/databases/workspace_mcp_servers.md` is created from this template on bootstrap and is **not tracked in Git**. Add optional userland MCP notes there (e.g. Scryfall, Mesen bridges). Do not commit machine-specific servers to the framework repo.

## Database (SQLite)

- **Server**: `Resonance7-sqlite` (Cursor may prefix `project-0-...` or `user-...`).
- **Use for**: Querying workspace SQLite databases (default: ingested session logs).
- **Default DB**: `library/databases/db/session_logs.db` (alias `session_logs`).
- **Other DBs**: Any `library/databases/db/<stem>.db` is reachable as alias `<stem>` (e.g. `iog_disassembly`). Discover via MCP tool `list_databases`.
- **Setup**: From the workspace root, run `python library/tools/scripts/setup_database.py`, then reload Cursor. Hand-edit from [`library/templates/mcp.json.example`](../templates/mcp.json.example) if needed.
- **Server code**: `library/tools/mcp_sqlite_server/` (`README.md`, `SETUP.md`).
- **More DBs**: See [`library/databases/README.md`](README.md) for layout, git policy, and ingest scripts.

## Optional userland MCP

- Configure additional servers in **`.cursor/mcp.json`** only (`setup_database.py` merges without removing your blocks).
- Document optional servers in **this local file** or under `library/docs/` — not in tracked framework templates or CHANGELOG unless you are shipping framework support.
